核心区别

普通工具是"一次性的问答机"。TechSpar 是一个闭环——它把每次训练的结果变成下次训练的依据。

Screen 2 / 6

5 个训练模式，同一个大脑

专项训练、简历面试、JD备面、实时Copilot、录音复盘——看起来是五个独立页面，实际上共享同一套记忆系统。

🎯

专项强化训练

针对某个技术主题（如图神经网络、系统设计等）反复练习，直到掌握。

📄

简历模拟面试

AI 读取你的简历，针对项目经历深挖细节——比如"这个项目用了什么架构？为什么这样设计？"

🎯

JD 定向备面

粘贴目标岗位的JD，系统分析该岗位所需技能，主动安排相关问题。

⚡

实时 Copilot

在真实面试中打开，实时识别面试官追问方向，提前预警你可能答不好的点。

🎙️

录音复盘

上传面试录音，自动转写并分析薄弱点，写入你的长期记忆，供下次训练使用。

ℹ️
关键洞察

这五个模式不是"五个不同的工具"，而是同一个 Profile的五个不同入口——它们共享同一份数据，通过统一的 API 读写记忆——它们一起构成了 TechSpar 的完整训练闭环。

底层代码说了什么？

以下是 backend/main.py 中的核心导入语句——它们连接了五大训练模式与记忆系统的关键管道。

CODE 
from backend.graphs.resume_interview import create_resume_interview_graph
from backend.graphs.topic_training import create_topic_training_graph
from backend.memory import get_profile_summary, update_profile
from backend.spaced_repetition import get_due_reviews, sm2_update
PLAIN ENGLISH

Bring in the resume interview graph (a state machine that runs the resume interview conversation).

Bring in the topic training graph — handles focused practice on a specific subject.

Bring in two memory functions: one reads the user's profile, the other writes updates to it.

Bring in the spaced repetition engine: get_due_reviews finds what you should review today, and sm2_update records your new performance level so the algorithm knows when to ask again.

ℹ️
关键组件一览

这四行代码揭示了 TechSpar 的核心架构：LangGraph 负责训练流程编排，backend.memory 负责读写用户记忆，SM-2 算法负责决定"什么时候该复习什么"。
