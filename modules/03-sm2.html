3-2 SM-2 的两个核心数字

SM-2 算法只用两个数字，就能量化你的记忆状态。

📅

interval_days

间隔天数——"多久之后再考你一次"。从1天起步，每次答对就翻倍：1天 → 3天 → 7天 → 14天……

答对：间隔 = 上次间隔 × 难度系数
⚙️

ease_factor

难度系数——"这道题到底有多难"。初始默认2.5，每次答题后根据表现微调，最低不低于1.3。

答难→高分 → 系数变大 → 间隔涨得更快
📊
state 是什么？

state 就是这两个数字加上一个 repetitions（连续答对次数）的组合，每次答题后 SM-2 用这三个数算出下一轮复习时间。这组数据存在 JSON 里，叫 sr_state。

3-3 逐行拆解：sm2_update 函数

这是 TechSpar 后端的实际代码，逐行翻译成大白话。

CODE 
def sm2_update(sr_state: dict, score_0_10: float) -> dict:
    quality = min(5, int(score_0_10 / 2))
    # 0-10分 → SM-2质量0-5
    ef = sr_state.get("ease_factor", 2.5)
    # 难度系数，最小1.3
    reps = sr_state.get("repetitions", 0)

    if quality >= 3:  # 通过
        if reps == 0: interval = 1
        elif reps == 1: interval = 3
        else: interval = int(sr_state.get("interval_days", 1) * ef)
        reps += 1
    else:  # 失败，重置
        interval = 1
        reps = 0

    ef = max(1.3, ef + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02)))
    return {
        "interval_days": interval,
        "ease_factor": round(ef, 2),
        "repetitions": reps,
        "next_review": (date.today() + timedelta(days=interval)).isoformat(),
        "last_score": score_0_10,
    }
PLAIN ENGLISH

Define a function that takes the current memory state of a question and a score (0-10), then returns the updated state for the next review session.

Convert the 0-10 score into SM-2's internal "quality" scale (0-5) by dividing by 2 and capping at 5. A 10/10 score becomes quality=5.

-

Look up the ease_factor from the current state. If there's no record yet, default to 2.5 — the standard starting difficulty for any new question.

-

Also look up how many times in a row this question has been answered correctly. Start at 0 if it's a fresh question.

-

If the quality is 3 or higher (answered correctly enough): figure out the next review interval.

First time correct → review again in 1 day.

Second time correct → now 3 days.

Third+ time → multiply the previous interval by the ease_factor. This is the compounding magic: each success stretches the interval further.

Increment the consecutive-correct counter by 1.

If the quality is below 3 (failed or shaky): reset the interval to 1 day and start the counter over from zero.

Interval goes back to 1 day.

Counter resets to 0.

-

After all this, update the ease_factor using a formula based on how well the question was answered. Never let it drop below 1.3.

-

Return all four updated values so the system knows when to bring this question back.

interval_days — how many days until the next review.

ease_factor — updated difficulty coefficient (rounded to 2 decimal places).

repetitions — how many times answered correctly in a row now.

next_review — the actual calendar date for the next review, calculated as today + interval_days.

last_score — the score you just gave, stored for the system's records.

关键洞察

SM-2 用两个简单的规则模拟了记忆的衰减规律：答对说明记忆还牢固，多给几天让它自然淡忘再回来考；答错说明记忆已经不牢，就不要硬拖时间，立刻回来救你。

如果你下次回来答错了，我会立刻把间隔重置回1天。所以你的记忆永远不会真正丢失——我只是在"快要忘"和"来得及巩固"之间找最佳平衡点。

S
SM-2 算法
